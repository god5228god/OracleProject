○ 1. 로그인 + 로그인 프로시저
- 관리자 메인 목업 화면
로그인 유형: 관리자 / 교수 / 학생 프로시저 내부 분기
PRC_AD_LOGIN → ADMIN 테이블
PRC_PF_LOGIN → PROF 테이블
PRC_ST_LOGIN → STUD 테이블
로그인 성공 시 사용자 ID 반환
이후 접근 가능한 메뉴 결정


○ 2. 관리자 모드 구현도 
- 관리자 메인 목업 화면
탭 구조:
•교수자 관리
•학생 관리 (생성은 최종관리자 ONLY)
•관리자 관리 (최종관리자 ONLY)
•과정 관리 (최종관리자 ONLY)
•과목 관리
•개설과정 관리
•개설과목 관리
•성적 관리 



○ 3. 교수자 등록 (관리자)
-교수자 등록 목업 화면
•PRC_PF_C 실행 결과
•INSERT 후 조회 결과(또는 VIEW)
 ▷ 시나리오 3-1. 교수자 수정·삭제
    수정은 보여주고, 삭제는 정책 설명 위주로
    •실제 삭제는 FK 연쇄 이슈 설명
    •물리 삭제의 위험성 설명



○ 4. 교수자 모드 구현도 
- 교수자 메인 목업 화면
탭 구조:
•성적 관리 (입력, 수정)
•과목 관리 (수정 ONLY - 배점)


○ 5. 교수 성적 입력 (교수자)
- 교수자 성적 등록 목업 화면
•강의 과목 목록 뷰
•프로시저 차단 결과 캡처(본인과목아니면 권한없음)

DB 정책:
•과목 종료일 이후만 성적 입력 가능
•중도탈락자는 제외(과목종료일기준 이전에 중도탈락일자일때)


○ 6. 개설과목 배점 입력(교수자)
- 교수자 개설과목 수정 목업 화면
• 배점 입력 테스트
• 배점의 합계가 100 미만이나 초과일 때의 예외 처리 


○ 7. 학생 성적 조회
- 학생 성적 조회 목업 화면
7-1. 일반 학생
•성적 VIEW 결과
•완료된 과목만 출력
•진행 중 과목 미출력 설명

7-2. 중도탈락 학생
•탈락 전 종료 과목 성적만 출력
•탈락 후 과목 미출력


○ 8. 과목 삭제
•삭제 시도 → 프로시저 차단
정책 설명:
•개설과정 시작 후 삭제 불가
•성적/수강 이력 존재 시 삭제 불가


○ 9. 관리자 수강신청 등록
•학생 대신 관리자 수강신청
•시작된 과정 차단
