시나리오)
1. 로그인 + 로그인 프로시저
로그인에는 관리자, 교수자, 학생 로그인이 분리가 되어있고 각 로그인 환경에 따른 어떤 테이블에서 id, pw를 가져와 확인하는지를 프로시저 실행 결과를 오른쪽에 두고 목업 페이지를 왼쪽에 넣어서 구성하려고 생각중.

2. 관리자 모드 구현도(최고 관리자로 로그인했을 경우를 상정)
관리자 모드에 있는 교수자 관리, 과정 관리, 과목 관리, 학생 관리, 성적 관리 탭이 있는걸 보여 주는게 어떨지 생각만?

3. 최고 관리자에서 교수자 입력(교수자 관리에서 입력을 선택 했을 때)
교수자의 이름, 주민번호를 입력하는 목업 페이지 + 프로시저 실행 결과
와 교수자의 이름 주민번호를 통해 입력된 교수자ID와 PW, 주민번호, 이름이 출력 되는 화면과 뷰를 통한 결과물 보여줘야 함.

3-1. 관리자에서 교수 수정 및 삭제(이걸 근대 구분해야 할까? 일단 한다면)
교수자 관리에서 등록된 교수들의 정보들이 뜨며 수정 및 삭제를 하는 페이지를 보여주고 거기서 수정을 눌렀을 때 교수자의 수정 정보를 입력 했을 때
교수자 정보 수정이 이루어 진걸 보여줘야함. 

4. 교수에서 성적을 입력하는 페이지 
교수에서 성적을 처리 할 때 보여 지는 목업 화면의 차이(강의중인 과목의 종료일을 보고 성적입력이 가능?)을 이건 자바에서 처리를 해야 할 것 같은데 일단 대략적 인건 (성적입력 버튼 클릭 → 과목명 선택 → 선택이 안됨) 이런 식으로 가야하는데 이때 버튼이 선택이 안 되는걸 (OPEN_SUB 테이블 기준 OS_EDATE와 SYSDATE를 활용하여 자바에 RETURN을 해야하는데)
★ 이걸 구현을 해? 이거 DB 프로젝튼대? 그렇다고 구현을 안해? 이거 목업을 자바라 연동해서 하는 것처럼 작업을 해놓고?

일단 저걸 넘긴다고 한다면
뷰를 통해 중퇴자가 그 개설과정에 있는 것과 성적입력에서는 안보여지는 차이를 보여줘야 함.

5. 교수에서 배점을 넣을때?
1. 교수가 배점의 합이 100점이 안되게 넣는 경우
프로시저에서 에러 메세지가 출력되는 화면을 보여준다.

2. 교수가 배점을 정상적으로 넣었을 때
프로시저에서 정상 작동하여 성적의 배점이 들어간 걸 보여준다.

6. 학생에서 성적이 출력된 화면
1. 일반 학생의 경우
자신이 수강한 성적이 출력되는 걸 뷰로 보여주며
수강중인 과목의 경우 출력이 안 돼는 것도 보여 줘야함?

1-2. 성적이 나온 과목의 수가 다른 학생들을 비교하는 페이지 필요.
여러 개의 과목을 수강한 학생인 경우는 모든 과목을 한 페이지에서 확인할 수 있어야 한다.

2. 중퇴자의 경우
자신이 수강한 과목에 대한 성적 점수가 출력되는걸 보여 줘야함.

7. 과목을 삭제 하는 페이지
DELETE 프로시저를 어떤 식으로 구성하고 처리하였는지 보여줘야 할 것 같음 + 권한에 대한 처리 코드가 있어서 발표에서 내용 보강이 가능함.


6. 개설과정 생성예시와 프로시저
 개설과정 정책설명 개설과정 등록,수정,삭제 (중복되는 과정을 개설할수없다,중복되는 강의실로 등록할수없다, 등)
관리자 개설과정 등록 목업 예시와 등록에 대한 프로시저 실행결과 확인
등록된 개설된 과정전체를 보여주는 뷰 확인

7. 개설 과목 수정예시와 프로시저
 개설 과목 등록,수정,삭제 에 대한 정책설명(등록시 중복되는 시간대,강사등을 등록할수없다,개설과정의 시작일이지나면 삭제불가능 등)
관리자 개설 과목 수정 목업 예시, 프로시저 실행결과 확인
